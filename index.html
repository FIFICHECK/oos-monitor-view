📖 OOS Monitor 使用手冊

📌 什麼是 OOS Monitor？

自動檢查 HKTVmall 產品庫存狀況，每 30 分鐘監測一次。支援以下狀態：

🔑 使用須知

🔒 唯讀版本 — 此 Dashboard 僅供監察庫存狀況。
如需管理 SKU（新增/刪除/Edit 監測期間），請前往 主 Dashboard。

  • ✅ In Stock — 庫存 ≥ 30，有「加入購物車」按鈕
  • 🟡 Low Stock — 庫存 10-29
  • 🔴 Super Low — 庫存 < 10
  • 🔴 OOS — 缺貨（暫時無法購買 / 已售罄）
  • ⏳ Pending — 未到監測開始日期
  • ⏰ Expired — 已過監測結束日期（移至歷史）

📊 Dashboard 區域說明

  • 頂部 Header — 標題、監測期間倒數。下方可同時顯示 🔴 OOS / 🔴 Super Low / 🟡 Low Stock 3 個獨立 badges（閃爍），全部正常時顯示 ✅ All in Stock
  • 跑馬燈 — 自動滾動顯示 OOS / Super Low / Low Stock 產品
  • 統計卡片 — Active / Pending / In Stock / Low / Super Low / OOS / Last Checked
  • 表格區域 — Out of Stock → Super Low → Low Stock → All SKUs → Pending → Expiry SKUs
  • 設定面板 — 底部可展開，管理 SKU 清單

➕ 新增 SKU

⚠️ 重要：加 SKU 之前，請先喺 HKTVmall 開產品頁確認正確嘅 SKU ID（URL 最後一組數字），確保係你想要監測嘅產品。

方法一（單個）：

  1. 展開底部 ⚙️ Settings 面板
  2. 喺 Add SKU 輸入產品 ID（格式：B0961005_S_4892218083908）
  3. 可選設定 Start 日期時間（監測開始時間）及 End 日期時間（監測結束時間）
  4. 按 ➕ Add SKU 按鈕
  5. 系統會自動 trigger 即時檢查

方法二（批量）：

  1. 喺 Bulk Add SKUs 區域，每行一個 SKU 貼上
  2. 可選設定批量 Start/End 時間
  3. 按 📋 Bulk Add 提交

🗑️ 移除 SKU

Settings 嘅 SKU List 而家分拆成兩個表：

  • 🟢 Active SKUs — 未過期嘅 SKU（active + pending）
  • ⏰ Expired SKUs — 已過 End Date 嘅 SKU

每個 SKU 右邊有個 ✕ 按鈕，點擊即刪除。

亦可剔選多個 SKU（右上角 全選 剔選全部），然後按 🗑️ Delete Selected 一次過批量刪除。兩個表各自有獨立嘅 batch delete。

📜 Status History

Click 任何表格中嘅 SKU 編碼（藍色連結）即可彈出視窗睇該 SKU 嘅狀態變化紀錄。所有表格（OOS、Super Low、Low Stock、All SKUs、Expiry SKUs）嘅 SKU 都支援 click 睇歷史。

  • Active SKU：顯示所有紀錄（由開始監測至今）
  • Expired SKU：只顯示 監測期間內（Start ~ End）嘅紀錄
  • 每個 SKU 保留最近 1000 條紀錄
  • Super Low 同 Low Stock 嘅 tag 會顯示當時庫存數，例如 🔴 Super Low (3)

📤 Export to Excel

每個 table 右上角有個 📥 Export XLSX 按鈕，一鍵匯出該 table 做 Excel。

🔍 搜尋功能

每個 table 右上方都有搜尋框，輸入關鍵字即可即時 filter。搜尋範圍包括 SKU、Product Name、日期（Start / End Date）。

🚀 Save & Run Now

Settings 底部有個綠色 「🚀 Save & Run Now」 掣。Click 後會立即透過 GitHub Actions 執行一次完整 check，唔使等 30 分鐘 cron job。

  • 加完 SKU 後唔想等 -> Click 🚀 掣
  • 大約 2-3 分鐘後完成
  • 旁邊嘅 🔄 Refresh Page 掣可以即時 reload Dashboard

🔔 Discord 自動通知

系統會自動監測 OOS、Super Low Stock 同 Low Stock 嘅新 SKU，並 Send 通知到 #general 同 check OOS status thread：

  • 🟡 每個 SKU 第一次變 Low Stock -> Send 1 次
  • 🔴 惡化到 Super Low -> Send 1 次
  • 🔴 再惡化到 OOS -> Send 1 次
  • 同一 status 唔會重複 send

⏱️ 自動更新

  • Dashboard 每 30 秒 自動 refresh 畫面
  • 後端 cron job 每 30 分鐘 檢查一次產品庫存
  • 右上角有 Next Cron 倒數計時器

💡 小貼士

  • 如果啱啱加咗 SKU 但未見到，等 1-2 分鐘後 Ctrl+F5 硬刷新 頁面
  • 設定 Start/End 時間可以精準控制每個 SKU 嘅監測窗口，唔使 manual 加減
  • 手機瀏覽可以左右滑動睇更多欄位（Horizontal Scroll）
  • Dashboard 數據存在 GitHub，清除 browser cache 唔會影響數據
